# RTMPose Mobile Vision + ROS2

A vision system based on **RTMPose** for detecting human pose from a mobile phone camera, computing approximate external anatomical landmarks, and publishing the results through both **HTTP** and **ROS2**.

This project has been tested in a very specific environment. Due to compatibility constraints between PyTorch, CUDA, MMPose, MMCV, and the rest of the dependencies, **it is recommended to reproduce the environment as faithfully as possible**. The main versions used in this project are: Python 3.8.20, PyTorch 2.1.0, torchvision 0.16.0, mmcv 2.1.0, mmengine 0.10.7, mmpose 1.3.2, OpenCV 4.13.0, and Flask 3.0.3.

---

## Features

- Receiving images from a mobile app over HTTP.
- Human pose processing with RTMPose.
- Computation of approximate external anatomical landmarks:
  - neck base
  - estimated thyroid
  - pelvis
  - estimated prostate
- Real-time visualization of the processed image through MJPEG.
- Export of landmarks in JSON format.
- Publication of image, landmarks, and keypoints in ROS2.

---

## Architecture

The system is divided into two main parts.

### 1. Vision server

It receives frames from the mobile device, runs RTMPose, computes landmarks, and generates:

- a processed image,
- a JSON file with landmarks and keypoints,
- an MJPEG stream for visualization.

### 2. ROS2 publisher

It reads the output generated by the server and publishes it as ROS2 topics.

---

## Main scripts

### `landmarks_unified.py`
Main vision module. It is responsible for:

- running RTMPose,
- selecting the best detected person,
- computing derived landmarks,
- estimating approximate body orientation,
- drawing the results on the image.

### `mobile_pose_server.py`
Flask server that:

- receives images from the mobile app,
- processes each frame,
- saves the latest results to disk,
- exposes HTTP endpoints for image, stream, and landmarks.

### `publish_mobile_pose_topics.py`
ROS2 node that:

- reads the processed image and the generated JSON file,
- publishes the results to different topics.

### `pose_server_discovery.py`
Auxiliary script used by the server to make discovery on the local network easier.

## Environment setup

To reproduce the vision environment, first create the base Conda environment and then manually install the additional required packages using `python -m pip` and `python -m mim`.

```bash
conda create --name openmmlab --file openmmlab_explicit.txt
conda activate openmmlab
```

### Install pip inside the environment

```bash
conda install pip setuptools wheel -y
```

To check that `pip` is associated with the active environment, you can run:

```bash
which python
which pip
python -m pip --version
```

The Python interpreter and `pip` should point to the Conda environment `openmmlab`.

Important: even if the environment is activated, it is recommended to always use `python -m pip` instead of plain `pip`, to make sure the installation is performed inside the correct environment.

### Install OpenMIM

```bash
python -m pip install -U openmim
```

### Install MMEngine and MMCV

```bash
python -m mim install mmengine==0.10.7
python -m mim install mmcv==2.1.0
```

### Install the remaining required dependencies

```bash
python -m pip install mmpose==1.3.2 mmdet==3.2.0 flask==3.0.3 opencv-python==4.13.0.92 addict==2.4.0
```

## Running the system

Go to the project folder and open several terminals.

### Terminal 1: vision server

```bash
conda activate openmmlab
python3 mobile_pose_server.py
```

### Terminal 2: ROS2 publisher

```bash
source /opt/ros/humble/setup.bash
/usr/bin/python3 publish_mobile_pose_topics.py
```

### Terminal 3: topic check (optional)

```bash
source /opt/ros/humble/setup.bash
ros2 topic list
ros2 topic echo /pose_app/thyroid
```

For example, the topic `/pose_app/thyroid` can be inspected.

## Using the mobile application

An Android app with automatic IP detection is included.

To use it:

1. Install Android Studio.
2. Open the app project.
3. Build it and install it on the phone.
4. Make sure the phone and the computer are connected to the same Wi-Fi network.

### Important note about networking

It has been observed that directly sharing mobile data from the phone to the computer does not always work properly in this case. The recommended setup is:

- connect both devices to the same Wi-Fi network,
- or use a third device to share internet with both.

If the IP search screen is accidentally closed, it can be opened again using the **Buscar PC** button. The other buttons can be ignored if you only want to test the vision system, as they are intended for manual robot control.

## Connection check

Once the scripts are running, you can check from the computer browser that the server is active:

```sh
http://IP_OF_THE_PC:8090/health
```

Example:

```sh
http://10.65.42.217:8090/health
```

If the server is running correctly, it should return a response indicating that the service is active.

To view the processed image in the browser:

```sh
http://IP_OF_THE_PC:8090/stream.mjpg
```

This stream allows you to visually verify that inference is running correctly and that the processed image is reaching the server.

## Available HTTP endpoints

```text
GET /health
```

Checks whether the server is active.

```text
GET /stream.mjpg
```

Displays the MJPEG stream with the processed image.

```text
GET /frame.jpg
```

Returns the latest processed image.

```text
GET /landmarks
```

Returns the latest JSON file with landmarks and keypoints.

```text
POST /upload_frame
```

Receives a JPEG frame from the mobile application.

## Published ROS2 topics

The `publish_mobile_pose_topics.py` node publishes the following topics:

- `/pose_app/image/compressed`
- `/pose_app/thyroid`
- `/pose_app/prostate`
- `/pose_app/keypoints`
- `/pose_app/all_landmarks`
- `/pose_app/debug`
- `/pose_app/orientation`
- `/pose_app/measurement_allowed`

## Important notes

### Execution order

`mobile_pose_server.py` must be started first, since `publish_mobile_pose_topics.py` depends on the files generated by the server inside `mobile_output/`.

### GPU

The system is currently configured to use the GPU through:

```sh
device="cuda:0"
```

If you want to run it on CPU, the corresponding code must be modified.

### ROS2

The ROS2 script is executed with:

```sh
/usr/bin/python3 publish_mobile_pose_topics.py
```

This means ROS2 must be properly installed on the system, not only inside the Conda environment.

## Troubleshooting

### The mobile app cannot find the PC

- Check that both devices are connected to the same Wi-Fi network.
- Check that the firewall is not blocking port `8090`.
- Verify that the server is running by accessing `/health` from the PC browser.
